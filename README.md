# blizzard-openapi

An unofficial OpenAPI 3.1 specification for Blizzard's Battle.net APIs.

Blizzard does not publish a machine-readable API specification. This project aims to provide one, built by:

1. Extracting endpoint paths, parameters, and authentication requirements from the [blizzardapi3](https://github.com/lostcol0ny/blizzardapi3) Python wrapper.
2. Cross-checking coverage against Blizzard's official HTML documentation.
3. Capturing live response payloads and inferring JSON Schema for each endpoint.
4. Merging everything into a single validated `openapi.yaml`.

## Status

Early work in progress. Not yet usable.

## Scope

Covers the following Blizzard APIs:

- World of Warcraft — Game Data, Profile, Media, Search (retail and Classic)
- Diablo 3 — Game Data and Community
- StarCraft 2 — Game Data and Community
- Hearthstone — Game Data
- OAuth 2.0 flows (Client Credentials, Authorization Code, OpenID Connect)

## Layout

```
openapi.yaml         # base spec (info, servers, tags, security, shared parameters)
paths/               # per-domain path entries (auto-generated by scaffold_paths.py)
  wow_game_data.yaml
  wow_profile.yaml
  wow_classic.yaml
  d3.yaml
  sc2.yaml
  hearthstone.yaml
schemas/             # per-domain component schemas (populated by live-capture phase)
scripts/             # build tooling
  scaffold_paths.py  # extracts paths from blizzardapi3 source
  bundle.py          # merges openapi.yaml + paths/ + schemas/ -> dist/openapi.yaml
samples/             # raw API responses (gitignored)
dist/                # bundled output (committed, for direct consumption via raw URL)
  openapi.yaml
```

## Build

```bash
# 1. Regenerate paths/*.yaml from blizzardapi3 source
uv run --with pyyaml python scripts/scaffold_paths.py

# 2. Cross-check coverage against Blizzard's HTML docs
uv run --with pyyaml python scripts/coverage_report.py

# 3. Capture live response samples (requires BLIZZARD_CLIENT_ID / _SECRET)
uv run --with httpx python scripts/capture_samples.py

# 4. Infer JSON Schemas from samples/
uv run --with pyyaml python scripts/infer_schemas.py

# 5. Bundle into dist/openapi.yaml (wires schemas into path responses)
uv run --with pyyaml python scripts/bundle.py

# 6. Validate
uv run --with openapi-spec-validator python -c \
  "from openapi_spec_validator import validate; \
   from openapi_spec_validator.readers import read_from_filename; \
   spec, _ = read_from_filename('dist/openapi.yaml'); validate(spec); print('ok')"

# 7. Lint with Redocly (honors redocly.yaml)
npx --yes @redocly/cli lint dist/openapi.yaml

# 8. Render HTML preview
npx --yes @redocly/cli build-docs dist/openapi.yaml -o redoc-static.html
```

Steps 1 and 5 are the minimum to regenerate the spec from source. Steps 3 and 4 are needed when blizzardapi3 adds new endpoints or when we want to refresh response schemas.
